# n8n Render Deployment

This repository contains the configuration files needed to deploy n8n on Render.

## Quick Start

1. Fork or clone this repository
2. Connect your GitHub repository to Render
3. Render will automatically detect the `render.yaml` file and configure your n8n instance
4. Update the environment variables in `render.yaml` with your secure values

## Configuration

### Environment Variables

The `render.yaml` file is pre-configured with:
- `N8N_ENCRYPTION_KEY`: Automatically generated by Render for data encryption
- `WEBHOOK_URL`: Automatically set to your Render service URL
- `N8N_PROTOCOL`: Set to HTTP (Render handles HTTPS/TLS termination at the load balancer level)

### Security

⚠️ **IMPORTANT**: The Dockerfile contains default basic authentication credentials (admin/yourpassword). These MUST be changed before deployment!

**Recommended approach**: Override these in `render.yaml` by adding:
```yaml
- key: N8N_BASIC_AUTH_USER
  value: your_secure_username
- key: N8N_BASIC_AUTH_PASSWORD
  value: your_secure_password
```

Alternatively, use Render's secret management to set these values securely.

## Files

- `Dockerfile`: Docker configuration for n8n
- `render.yaml`: Render service configuration

## Deployment

Once connected to Render, your n8n instance will be available at your Render subdomain with basic authentication enabled.
